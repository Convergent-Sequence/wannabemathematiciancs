Análisis Teachable Machine

Creación de las clases

En esta sección se especifican las diferentes clases con las cuales contará el modelo. Cada clase se debe dotar con muestras, es decir, datos mediante las cuales el modelo se entrenará para reconocer datos del mismo tipo posteriormente. La colección de las muestras de cada clase se denomina conjunto de datos de preparación.

Preparación del modelo

En esta sección se entrena el modelo para poder probarlo, tiene las siguientes opciones ajustables:
1. Épocas. Es la cantidad de veces que el modelo procesa el conjunto de datos de preparación; en general, a mayor número de épocas el modelo realiza mejores predicciones.
2. Tamaño del lote. Si se tiene una cantidad k de lotes, se dividen los datos de cada clase en lotes de tamaño k, de modo que el modelo completa una epoca cuando procese los k lotes de cada clase.
3. Tasa de apredizaje. En términos simples, la tasa de aprendizaje le dice a ciertos vectores gradientes con que velocidad moverse para alcanzar máximos y mínimos, es decir, afecta la velocidad con la que el algortimo se vuelve más óptimo.
Cambiar la tasa de aprendizaje puede dañar el modelo.

Vista previa y exportación del modelo

Se prueba el modelo y se verifica que funcione correctamemte para exportarlo. Se puede exportar en formato javascript o TensorFlow lite para ser probado en la app de TensorFlow. Se puede descargar o subir a la nube, en esta última opción, tensorFlow llama al modelo a través de un link que se puede implementar en otros proyectos, siempre y cuando se tenga acceso a internet.
En el caso de los modelos de sonido, al exportarse el modelo, se puede elegir el factor de superposición del mismo, este factor determina la frecuencia con la cual el último segundo de audio se compara con el modelo.
